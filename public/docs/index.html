本地预览

将 HTTP 服务器的文档根目录指向本仓库的 public 文件夹。示例（Python 内置服务器）：

cd public && python3 -m http.server 8080

在浏览器打开 http://127.0.0.1:8080/。所有页面使用以 / 开头的资源路径，因此需要 Web 服务器，而不是直接双击打开 HTML 文件。

工具路由生成

路由与页面元数据的单一来源是 site/routes.yaml（slug、类型、标题与描述）；页面骨架由 site/templates/ 下的 Jinja2 模板渲染。英文根路径与 17 个语言前缀下的镜像路径均由构建脚本展开。

依赖安装（首次）：pip install -r requirements.txt

生成静态页：

python3 scripts/generate_pages.py

输出 public/index.html（来自 site/templates/home.html）、public/<slug>/index.html、public/<locale>/<slug>/index.html，以及 public/<locale>/index.html、public/<locale>/blog/index.html，并刷新 public/sitemap.xml。public/blog/ 下除生成器写入的镜像外，你仍可手工维护文章页。

新增工具或模型：只在 site/routes.yaml 的 pages 里增加 slug（kind: studio、mode、可选 category / hub_section），/ai-tools/ 分区与分类页列表会自动更新。说明见仓库内 site/ADDING_MODELS.md。

生成页包含 canonical、hreflang、Open Graph / Twitter 卡片占位、以及轻量 JSON-LD。域名与品牌字段在 site/routes.yaml 的 site 与 sitemap.base_url。

核对参考站 URL 清单：python3 scripts/fetch_sitemap_routes.py。与 heyvid 英文 slug 集合对账：python3 scripts/verify_sitemap_parity.py。

子域映射

法律与道德上不应复制第三方站点；本仓库提供的是 Sota Video Gen（品牌词：sota video gen）的原创页面。若你希望「主站 / 应用 / 博客 / 文档」分别用子域展示，可采用以下常见做法：

  • 同一静态目录 + 多主机名：在 CDN 或反向代理（如 Nginx、Cloudflare Workers）上为 app.example.com、blog.example.com 配置独立源站路径，分别映射到 /app/、/blog/、/docs/，并把该路径设为文档根或添加重写规则。
  • 多仓库或多构建产物：每个子域部署独立项目，仅共享设计系统；适合应用与营销站技术栈不同的情况。

本站页面链接已按路径划分：/ 营销首页、/ai-tools/ 工具目录、各工具子路径（如 /image-to-video/）、可选语言前缀（如 /ja/ai-tools/）、以及 /app/、/blog/、/docs/，便于你对照配置 DNS 与路由。
